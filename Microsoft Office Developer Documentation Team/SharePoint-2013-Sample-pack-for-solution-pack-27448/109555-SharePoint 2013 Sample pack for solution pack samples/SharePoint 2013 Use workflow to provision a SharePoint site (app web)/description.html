Description of the sample

This sample demonstrates how to use a workflow to provision a SharePoint site by using the remote provisioning pattern and CSOM. The sample uses an app installed event to associate a remote event receiver with the custom list on the app web.

For a more detailed description of this sample, see Module 5 of the SharePoint 2013 and SharePoint Online solution pack for branding and site provisioning.

Prerequisites

This sample requires the following:

  • Visual Studio 2012 or Visual Studio 2013

  • Microsoft Office Developer tools for Visual Studio 2012 or Visual Studio 2013

  • A SharePoint 2013 development environment

  • Create on the host web an approval workflow and associated list identical to the ones deployed to the app web.

Configure the sample

Check the Handle App Installed property in the project property of the app for SharePoint project.
