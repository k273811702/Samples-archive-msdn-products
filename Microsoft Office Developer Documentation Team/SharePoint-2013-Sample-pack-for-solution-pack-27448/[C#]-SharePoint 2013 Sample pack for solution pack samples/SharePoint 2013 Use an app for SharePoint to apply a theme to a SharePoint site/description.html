This sample demonstrates how to use CSOM to apply a theme to a SharePoint site.

Description of the sample

This provider-hosted sample app for SharePoint demonstrates using the Client-Side Object Model (CSOM) to obtain information about and make changes to a design theme or Composed Look by uploading component files and using APIs to apply them. This pattern can be used to brand remote-provisioned sites of all types.

Deviations from good practices

The sample demonstrates how to use CSOM to obtain information about and make changes to a Theme or Composed Look by uploading component files and using APIs to apply them. It does not conform to all the good practices that should be used in a production app:

  • The app uses files stored on the local file system of a provider-hosted app for SharePoint and hardcodes references to them in client code. A robust app for SharePoint would use an alternative storage mechanism and configuration store.

  • The app lacks proper error and exception handling.

Prerequisites

This sample requires the following:

  • Visual Studio 2012 or Visual Studio 2013

  • Microsoft Office Developer Tools for Visual Studio

  • A SharePoint 2013 development environment

Configure the sample

Follow these steps to configure the sample:

  • Open the ApplyThemeApp.sln file in Visual Studio.

  • In Properties pane of Visual Studio, change the Site URL property. It is the absolute URL of your SharePoint developer site collection on Office 365. For example, "https://<my tenant>.sharepoint.com/sites/dev/"

  • Press F5 to begin debugging.

  • If necessary, authenticate to your developer site.

  • Trust the App.

Run and test the sample

  1. The apps startup page should open automatically in the browser. You can also load the app by clicking the Apply Theme Sample App from the Apps in Testing list.

  2. Click Get Theme Info to see details about the current site look. Note the highlighted properties.

  3. Click Apply New Theme to invoke the ApplyTheme method. Check the status box for any errors or warnings.

  4. Again click Get Theme Info to view properties after theme application. Note updated values.

  5. Return to the home page of your developer site. Note that style updates have been applied - all fonts are Times New Roman as opposed to the default Segoe UI and hyperlink colors have been modified from the default blue.

  6. Examine the Composed Looks library by navigating to Site Settings and Composed Looks. Note that an item has been added for our newly-deployed theme.

    Because the Change the Look page is driven by this list, our theme is now available for site owners if they want to apply a different look and feel to the site.
