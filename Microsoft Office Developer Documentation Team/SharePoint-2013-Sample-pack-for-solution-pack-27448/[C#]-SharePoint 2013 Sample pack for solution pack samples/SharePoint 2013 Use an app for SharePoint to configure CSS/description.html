This sample demonstrates how to use a .NET client-side object model (CSOM) custom action to apply CSS to a SharePoint site.

Description of the sample

This auto-hosted app for SharePoint sample demonstrates using an app-installed event handler to brand a site using the client-side object model (CSOM). The sample uploads and applies a CSS file to all pages in a site using a custom action. This pattern can be used to brand all types of remote-provisioned sites.

Deviations from good practices

The sample demonstrates how to use an app-installed event and to apply a custom CSS file to a site, so it does not conform to all the good practices that should be used in a production app.

Note

The app uses files stored on the local file system of a provider-hosted app and hardcodes references to them in client code. A robust app would use an alternative storage mechanism and configuration store.The app lacks proper error and exception handling.

Prerequisites

This sample requires the following:

  • Visual Studio 2012 or Visual Studio 2013

  • Microsoft Office Developer Tools for Visual Studio

  • A SharePoint 2013 development environment

  • For debugging, a connection string for a Microsoft Azure Service Bus namespace is required. See How to: Debug a remote event receiver in an app for SharePoint 2013 for more details.

Configure the sample

Follow these steps to configure the sample:

  • Open the AlternateCSSAppAutohosted.sln file in Visual Studio.

  • In Properties pane of Visual Studio, change the Site URL property. It is the absolute URL of your SharePoint test site collection on Office 365. For example, "https://<my tenant>.sharepoint.com/sites/dev"

  • Open the Properties window for the AlternateCSSAppAutohosted project. Select the SharePoint section. Select Enable remote event debugging and enter your Microsoft Azure Service Bus connection string.

  • Add a breakpoint in the ProcessEvent method of the AppEventReceiver class (found in Services/AppEventReceiver.svc.cs).

  • If necessary, authenticate to your developer site.

  • Trust the app.

Run and test the sample

After configuration is complete, your browser will be redirected to the app's homepage, which confirms that custom actions have been applied to the host web.

  1. Click Return to Site.

    Note

    At this point, style changes have been applied. For example, the sample CSS file changes the color of all hyperlinks.

  2. Press F5 to compile and run the sample.

  3. Observe that the ProcessEvent breakpoint is hit, indicating the event has been fired and handled. If desired, step through the code. Otherwise, press F5 to continue.
