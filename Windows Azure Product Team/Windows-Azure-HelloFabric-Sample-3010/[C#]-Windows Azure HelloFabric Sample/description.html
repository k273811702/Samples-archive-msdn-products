Introduction

The Windows Azure HelloFabric sample is a simple hosted service that demonstrates a web role and a worker role and uses the Windows Azure Managed Library to interact with Windows Azure from running role instances.

The Windows Azure HelloFabric sample demonstrates the following features of the Windows Azure SDK:

  • Obtain information about the running service, which includes status, configuration settings, storage resources, and external and internal endpoints.
  • Log errors, messages, and performance counters, and then transfer the logs to a storage account.
  • Configure a web role to log Failed Request errors, and transfer those logs to a storage account. 
  • Configure a local storage resource and write a message to it.
  • Communicate between a web role and a worker role by using an internal endpoint.

You can use the following information to understand and run the HelloFabric sample:

  • Collecting Logging Data by Using Windows Azure Diagnostics
  • Enabling Communication for Role Instances in Windows Azure
  • Using the Windows Azure Storage Services
  • Windows Azure Tools

Prerequisites

  • Storage account – The sample is configured for using the Windows Azure Compute Emulator and diagnostic data is collected in a local storage account. If you want to use this sample with a subscription in Windows Azure, you must create a storage account in Windows Azure and provide the account information in the service configuration for the sample.
  • Windows Azure SDK – provides tools and files that are needed to complete the development process of an application for Windows Azure.
  • Visual Studio 2010 – the sample provides a Visual Studio 2010 project that you can use to deploy the sample.

Building and Running the Sample

To run the sample from Visual Studio 2010

  1. Open Visual Studio 2010 as an administrator.
  2. Browse to the folder where you extracted the sample and open HelloFabric.sln.
  3. Press F6 to build the solution.
  4. Press F5 to debug the solution. When you debug or run the application from Visual Studio, Visual Studio packages the application, starts the Windows Azure Compute Emulator, deploys the application to the Compute Emulator, and launches the browser to  display the default web page defined by the web role.

Configuring the diagnostic connection string

The Windows Azure Managed Library provides diagnostics classes to write log data to the Windows Azure storage services, either at scheduled intervals or by requesting an on-demand transfer. By default, the Windows Azure HelloFabric sample is configured to write log data to development storage. If you choose to deploy the application to Windows Azure, you must change the configuration string in the service model to specify a storage account in Windows Azure.

To change the diagnostic connection string in Visual Studio

  1. In Solution Explorer, expand Roles, right-click HelloFabric_WebRole, and then click Properties.
  2. On the Settings page, select Microsoft.WindowsAzure.Plugins.Diagnostics.ConnectionString, and then click the Edit (…) button.
  3. Select Enter Storage Credentials, and then enter your account name and key for your Windows Azure storage account.
  4. Select whether to use the default HTTP or HTTPS endpoints to connect to the storage services. Alternately, you can configure custom endpoints for each of the three storage services.
  5. Click OK.
  6. Repeat this process for the worker role.

To change the diagnostic connection string by using a text editor

  1. Edit the ServiceConfiguration.cscfg file using a text editor, such as Notepad.
  2. Change the value of the Microsoft.WindowsAzure.Plugins.Diagnostics.ConnectionString setting to use a connection string for Windows Azure storage. The connection string must be in the format shown in the example below.
  3. Locate the Microsoft.WindowsAzure.Plugins.Diagnostics.ConnectionString setting for the worker role and change it in the same way.

  4. Save the ServiceConfiguration.cscfg file.
