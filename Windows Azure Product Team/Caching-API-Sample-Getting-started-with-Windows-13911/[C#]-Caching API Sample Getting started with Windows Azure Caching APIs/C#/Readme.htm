Windows Azure AppFabric Caching API Usage Sample

This sample shows how to use various Windows Azure AppFabric Caching APIs.

Note: Samples are provided for educational purposes only. They are not intended to be used in a production environment and have not been tested in a production environment. Microsoft does not provide technical support for these samples.

Prerequisites

This sample assumes that the Windows Azure AppFabric SDK is installed and all cache related references are resolved in the sample's solution. The SDK can be installed from http://go.microsoft.com/fwlink/?LinkId=216119. The cache related binaries can be found in the following path:

<SDK install path>\V1.0\Assemblies\NET4.0\Cache 

Sample Flow

This sample exercises the following caching capabilities:

  1. Simple Add/Get to the default cache.

  2. Add/Get/GetAndLock/GetIfNewer/Put/PutAndUnlock to the default cache/region. This includes the following variations:

    1. GetAndLock trying to access a locked object.

    2. PutAndUnlock trying to unlock an object that is not locked.

  3. Add/Get/Put of a versioned object.

    1. Explicitly modify an item only if versions match.

    2. Fail to modify an item if a newer item is available in the cache.

Running the Sample

Before you run this sample, you must provision a cache on the Windows Azure AppFabric Management Portal. The properties window for your cache contains necessary information for connecting this sample. The cache Service URL must be placed in the PrepareClient() method in the Program.vb|Program.cs file. The cache Authentication Token must be put in either the PrepareClient() method or the app.config file. For more details on how to provision a cache on the portal, see http://go.microsoft.com/fwlink/?LinkId=202670.

When the executable is run, it will run through the flow described above and log its output to the console window.
